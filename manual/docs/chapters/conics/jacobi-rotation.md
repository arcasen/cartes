### Jacobi 旋转

在 Jacobi 旋转（用于对称矩阵对角化）中，为了避免直接计算耗时的三角函数（$\sin$ 和 $\cos$）以及反三角函数（$\arctan$），通常采用**代数方法直接计算旋转矩阵的元素 $c = \cos\theta$ 和 $s = \sin\theta$**。

核心思想是利用三角恒等式，通过矩阵元素直接解出 $c$ 和 $s$，或者先解出 $t = \tan\theta$，再推导 $c$ 和 $s$。

#### 问题背景

假设我们要消除对称矩阵 $A$ 中的非对角元素 $a_{pq}$（其中 $p < q$）。Jacobi 旋转矩阵 $J(p, q, \theta)$ 定义为：

$$
J_{ii} = 1 \quad (i \neq p, q) \\
J_{pp} = J_{qq} = c = \cos\theta \\
J_{pq} = -J_{qp} = s = \sin\theta
$$

旋转后的矩阵 $A' = J^T A J$ 要求新元素 $a'_{pq} = 0$。

根据矩阵乘法，$a'_{pq}$ 的表达式为：

$$
a'_{pq} = (c^2 - s^2)a_{pq} + cs(a_{pp} - a_{qq}) = 0
$$
令 $\tau = \dfrac{a_{qq} - a_{pp}}{2a_{pq}}$，上述方程可以重写为关于 $t = \tan\theta = \dfrac{s}{c}$ 的二次方程：

$$
t^2 + 2\tau t - 1 = 0
$$

#### 不计算三角函数的算法步骤

我们不需要计算 $\theta = \arctan(t)$，也不需要调用 `sin()` 或 `cos()`。只需按以下步骤计算 $c$ and $s$：

**第一步：计算辅助量 $\tau$**

$$
\tau = \dfrac{a_{qq} - a_{pp}}{2a_{pq}}
$$

**注意**：如果 $a_{pq} = 0$，则无需旋转。*

**第二步：求解 $t = \tan\theta$**

我们需要解 $t^2 + 2\tau t - 1 = 0$。为了保证数值稳定性（避免相近数相减导致精度丢失），我们选择绝对值较小的那个根：

$$
t = \dfrac{\text{sgn}(\tau)}{|\tau| + \sqrt{1 + \tau^2}}
$$

或者写作更常见的形式（若 $\tau \ge 0$，取负根；若 $\tau < 0$，取正根，以确保 $|\theta| \le \pi/4$）：

$$
t = \begin{cases} 
\dfrac{1}{\tau + \sqrt{1+\tau^2}} & \text{if } \tau \ge 0 \\
\dfrac{-1}{-\tau + \sqrt{1+\tau^2}} & \text{if } \tau < 0 
\end{cases}
$$

这可以统一写为：

$$
t = \dfrac{1}{\tau + \text{sgn}(\tau)\sqrt{1+\tau^2}}
$$

*(注：这里选择小根是为了让旋转角度 $\theta$ 落在 $[-\pi/4, \pi/4]$ 之间，保证 $c \ge s$，从而提高稳定性)*

**第三步：从 $t$ 计算 $c$ 和 $s$**

利用恒等式 $1 + t^2 = 1 + \tan^2\theta = \sec^2\theta = \dfrac{1}{c^2}$，我们可以直接得到：
$$
c = \dfrac{1}{\sqrt{1 + t^2}}
$$
$$
s = c \cdot t
$$

为什么这样做更好？

1.  **速度**：避免了昂贵的 `atan`、`sin`、`cos` 库函数调用。只需要一次平方根 (`sqrt`) 和基本的加减乘除。
2.  **精度**：
    *   直接计算 $\theta = \arctan(\dots)$ 然后求 $\sin(\theta)$ 会引入两次舍入误差。
    *   上述方法通过代数恒等式直接构造 $c$ 和 $s$，且通过选择小根策略（$|\theta| \le \pi/4$），确保了 $c$ 接近 1，$s$ 接近 0，这在浮点运算中是最稳定的区域。
3.  **更新矩阵元素**：
    一旦有了 $c$ and $s$，更新矩阵对角线和非对角线元素也只需要代数运算，无需再次涉及三角函数：
    *   $a'_{pp} = a_{pp} - t \cdot a_{pq}$
    *   $a'_{qq} = a_{qq} + t \cdot a_{pq}$
    *   $a'_{ip} = a_{ip} - s(a_{iq} + \tau a_{ip})$ （对于 $i \neq p, q$）
    *   $a'_{iq} = a_{iq} + s(a_{ip} - \tau a_{iq})$ （对于 $i \neq p, q$）
    *(具体的更新公式有多种等价形式，核心都是只用 $c, s, \tau, t$ 进行四则运算)*

这种方法就是经典的 **Golub-Kahan** 或 **Numerical Recipes** 中推荐的实现方式，完全避开了显式的三角函数角度计算。

---

为什么不写成公式：
$$
t = \begin{cases} 
-\tau + \sqrt{1+\tau^2} & \text{if } \tau \ge 0 \\
-\tau - \sqrt{1+\tau^2} & \text{if } \tau < 0 
\end{cases}
$$
**在数学上是完全正确的**，它确实是方程 $t^2 + 2\tau t - 1 = 0$ 的两个根中**绝对值较小**的那个根（即我们要找的 $\tan\theta$，其中 $|\theta| \le \pi/4$）。

然而，在**计算机数值计算**中，我们通常**不直接这样写**，而是写成我之前提到的倒数形式：
$$
t = \frac{1}{\tau + \text{sgn}(\tau)\sqrt{1+\tau^2}}
$$
原因在于**数值稳定性（避免“灾难性抵消”）**。

### 1. 什么是灾难性抵消？

当两个非常接近的数相减时，有效数字会大量丢失，导致结果精度急剧下降。

让我们看看你提出的公式在 $\tau$ 很大时会发生什么：

#### 情况 A：$\tau$ 是一个很大的正数 (例如 $\tau = 10^8$)
根据你的公式：
$$
t = -\tau + \sqrt{1+\tau^2} = -10^8 + \sqrt{1 + 10^{16}}
$$
由于 $10^{16}$ 远大于 $1$，$\sqrt{1+10^{16}} \approx 10^8$。
计算机在进行浮点运算时：
1.  计算 $\sqrt{1+10^{16}}$，结果可能是 $100000000.000000005$（取决于精度）。
2.  执行减法：$-100000000 + 100000000.000000005$。
3.  **问题**：这两个数的前 8-9 位数字完全相同。相减后，前面的有效数字全部抵消，只剩下最后几位可能充满舍入误差的数字。如果 $\tau$ 更大，结果甚至可能直接变成 **0**，这是完全错误的（因为 $t$ 应该约等于 $1/(2\tau)$）。

#### 情况 B：$\tau$ 是一个很大的负数 (例如 $\tau = -10^8$)
根据你的公式：
$$
t = -\tau - \sqrt{1+\tau^2} = 10^8 - \sqrt{1 + 10^{16}}
$$
同样的问题：两个巨大的正数相减，导致有效数字丢失。

### 2. 为什么倒数形式是安全的？

我们将你的公式进行有理化变形（分子分母同乘共轭）：
$$
-\tau + \sqrt{1+\tau^2} = \frac{(-\tau + \sqrt{1+\tau^2})(\tau + \sqrt{1+\tau^2})}{\tau + \sqrt{1+\tau^2}} = \frac{(1+\tau^2) - \tau^2}{\tau + \sqrt{1+\tau^2}} = \frac{1}{\tau + \sqrt{1+\tau^2}}
$$
注意分母变成了 **加法**：$\tau + \sqrt{1+\tau^2}$。

- 当 $\tau > 0$ 时，$\tau$ 和 $\sqrt{1+\tau^2}$ 都是正数，相加不会发生抵消，数值非常稳定。
- 当 $\tau < 0$ 时，我们使用另一支 $t = \frac{-1}{-\tau + \sqrt{1+\tau^2}}$，分母依然是两个正数相加。

### 3. 直观对比

假设 $\tau = 10^8$，真实值 $t \approx 0.5 \times 10^{-8}$。

*   **你的公式（直接相减）**：
    `float(1e8)` 和 `float(sqrt(1+1e16))` 在双精度下可能完全相等，或者最后几位随机。
    计算结果：`0.0` 或 `1.2e-15` (误差极大)。

*   **优化公式（倒数相加）**：
    分母：$10^8 + 10^8 = 2 \times 10^8$ (精确)。
    结果：$1 / (2 \times 10^8) = 0.5 \times 10^{-8}$ (精确)。

### 结论

你写的公式在**数学推导**和**符号运算**中是完全完美的，也是最简洁的形式。

但在**编写代码**（特别是使用浮点数 `float` 或 `double`）时，为了避免大数相减导致的精度崩塌，必须将其改写为**加法形式的倒数**。这就是为什么所有标准的数值线性代数库（如 LAPACK, NumPy, Numerical Recipes）都使用以下形式的原因：

```python
# 推荐写法 (数值稳定)
if tau >= 0:
    t = 1.0 / (tau + math.sqrt(1.0 + tau*tau))
else:
    t = -1.0 / (-tau + math.sqrt(1.0 + tau*tau))
```

这种技巧在数值分析中非常常见，被称为**有理化**或**避免相减抵消**。
